## [Unreleased]

### Added

- **Contract tests for replayt 0.5.x spike doc** — **`tests/test_replayt_0_5_spike_docs.py`** asserts `docs/REPLAYT_0_5_COMPATIBILITY_SPIKE.md` keeps required sections, lists bridge↔replayt API touchpoints, matches the `replayt` line from `pyproject.toml`, and stays linked from README, MISSION, DESIGN_PRINCIPLES, and ARCHITECTURE.
- **replayt 0.5.x compatibility spike (maintainer doc)** — **[docs/REPLAYT_0_5_COMPATIBILITY_SPIKE.md](docs/REPLAYT_0_5_COMPATIBILITY_SPIKE.md)** records PyPI status (no `0.5.x` as of 2026-03-24), rerun commands for when a pre-release or GA exists, bridge↔replayt API touchpoints, rough effort for follow-ups, and a draft **CHANGELOG** migration blurb. [docs/MISSION.md](docs/MISSION.md) adds refined acceptance criteria for widening the declared range; [docs/DESIGN_PRINCIPLES.md](docs/DESIGN_PRINCIPLES.md), [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), and README link the spike doc.
- **`runner_dry_run_plan` dry-check parity** — Optional MCP parameters **`strict_graph`**, **`metadata_json`**, **`experiment_json`**, and **`policy_hook_context_json`** (defaults preserve prior behavior: non-strict graph, other JSON arguments omitted) map through to **`replayt.cli.validation.validate_workflow_graph`** and **`validation_report`**, matching **`replayt run --dry-check`** knobs for integrators. Documented with JSON-schema-style input tables and a parity specification in **[docs/MCP_TOOLS.md](docs/MCP_TOOLS.md)**; **`tests/test_mcp_tools.py`** covers a happy path on the example target, invalid JSON, and **`strict_graph`** changing validation outcome on a synthetic workflow file.
- **Stdio MCP session smoke test** — [docs/MISSION.md](docs/MISSION.md) and [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) define refined acceptance criteria and architecture review for backlog **“Add an integration smoke test over the real stdio MCP session”**. **`tests/test_mcp_stdio_session_smoke.py`** now runs in default CI: MCP SDK `stdio_client` + `ClientSession`, `python -m replayt_mcp_bridge` with repo-root `cwd`, handshake-driven sync, `asyncio.wait_for` timeout, and **`replayt_version_info`** asserting structured `status: ok`.
- **Structured error specification** — [docs/MCP_TOOLS.md](docs/MCP_TOOLS.md) defines **`correlation_id`** on mapped `{ status: error, … }` tool results and the same value on structured stderr logs per invocation, plus an explicit exception/branch inventory and unmapped-exception rules. Cross-linked from [docs/MISSION.md](docs/MISSION.md), [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), and [docs/SECURITY.md](docs/SECURITY.md).
- **`correlation_id` on bounded tool errors** — `replayt_mcp_bridge.server` assigns one id per tool invocation (FastMCP `Context.request_id` when non-empty, else UUID4), attaches it to every mapped `_tool_error` payload, and emits the same value on `replayt_mcp_bridge.tool.begin` / `.end`, `replayt_mcp_bridge.tool.unhandled_exception`, and `replayt_mcp_bridge.store_hint.rejected` JSON log lines. Tests cover log/result alignment for a mapped path and correlation on unmapped exceptions before re-raise.
- **Optional `REPLAYT_MCP_BRIDGE_STORE_HINT_ROOTS`** — comma-separated absolute filesystem roots (parsed in `observability.py`); when set, **explicit** `store_hint` arguments to `persistence_list_run_events` must resolve under one of them. Omitted `store_hint` still uses replayt’s default log directory resolution (no default tightening). Documented in **[docs/SECURITY.md](docs/SECURITY.md)** with examples; rejections log **`replayt_mcp_bridge.store_hint.rejected`** without the client path string.
- **[docs/MCP_HOST_CONFIG.md](docs/MCP_HOST_CONFIG.md)** — copy-paste MCP host stdio configuration (**Claude Desktop** `mcpServers` and **Cursor** `.cursor/mcp.json` with **`type: "stdio"`**), **`replayt-mcp-bridge`** vs **`python -m replayt_mcp_bridge`**, Windows vs POSIX paths, and pointers to **[docs/SECURITY.md](docs/SECURITY.md)**.
